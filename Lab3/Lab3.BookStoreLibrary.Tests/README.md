# Лабораторная работа №3 по "Основам программирования"

## Технологичный стек
- Язвк программирования: C# с .NET 10
- Используемые технологии:
  - Windows Forms
- Архитектура: Многослойная (UI Layer и Business Logic Library)
  - Объектно-ориентированное программирование (инкапсуляция, полиморфизм, работа с коллекциями).

## Основные возможности
1. Управление фондом
- Добавление книг: Ручной ввод данных (название, автор, жанр, страницы, цена) или автоматическая генерация случайной книги на основе баз данных (текстовых файлов).
- Уникальность: Система автоматически проверяет уникальность названий и добавляет индекс (например, "Война и мир 2"), если книга с таким названием уже существует.
- Автоматическое распределение: Книги автоматически попадают на нужную полку в зависимости от жанра.
2. Структура магазина
- Жанровые полки (Bookshelf): Каждая полка имеет ограниченную вместимость (5 книг) и привязана к конкретному жанру.
- Динамическое управление: Если лимит жанровых полок (макс. 5) исчерпан, система предложит переназначить пустую полку под новый жанр.
- Учет баланса: Продажа книг увеличивает общий бюджет магазина в реальном времени.
3. Поиск и навигация 
- Гибкий поиск: Поиск книг в архиве по названию или по уникальному идентификатору GUID.
- Интерфейс: Многовкладочная система (TabControl) с блокировкой прямого переключения (навигация осуществляется только через кнопки управления для предотвращения ошибок ввода).

## Структура проекта
Проект разделен на два основных пространства имен:

### Бизнес-логика (Lab3.BookStoreLibrary)
- Book.cs: Модель данных книги. Содержит уникальный ID, характеристики и логику продажи.
- Bookshelf.cs: Контейнер для книг определенного жанра. Контролирует вместимость и поиск внутри полки.
- BookStore.cs: Главный управляющий класс. Хранит коллекцию полок, обрабатывает финансовые операции и управляет глобальным поиском.

### Интерфейс пользователя (Lab3.UI)
- BibleForm.cs: Логика взаимодействия с формой: обработка кликов, валидация ввода, обновление списков (ListView) и обработка исключений.
- BibleForm.Designer.cs: Автоматически сгенерированный код компоновки элементов управления.

### Тестирование (Lab3.BookStoreLibrary.Tests Unit Tests)
В проект включен отдельный блок тестов на базе xUnit, который гарантирует корректность работы бизнес-логики независимо от графического интерфейса.

#### Что покрыто тестами:
1. BookTest.cs:
- Проверка корректной инициализации свойств через конструктор.
- Валидация автоматической генерации уникального Id (GUID) при создании книги.
- Тестирование генератора случайных книг (GenerateRandomBook) на предмет соответствия диапазонам цен и страниц.

2. BookshelfTest.cs:
- Проверка логики добавления книг (нельзя положить книгу "Фантастика" на полку "Детектив").
- Тестирование лимитов вместимости полки.

3. BookStoreTest.cs:
- Тестирование финансовых операций (изменение баланса после продажи).
- Проверка логики управления полками в рамках магазина.

Чтобы запустить тесты, используйте команду в консоли:
```bash
dotnet test
```


## Как запустить
1. Убедитесь, что у вас установлена Visual Studio 2022 с нагрузкой ".NET Desktop Development".
2. Проверьте наличие необходимых текстовых файлов в директории запуска (или добавьте их в проект):
- Genre.txt — список доступных жанров.
- Authors.txt — база имен авторов.
- BookName.txt — база названий книг.
3. Скомпилируйте и запустите проект (F5).

```bash
dotnet run --project ./Lab3/Lab3.UI/
```

## Обработка ошибок
В приложении реализована продвинутая система обработки исключений (ShowError):
- Validation: Проверка на пустые поля при добавлении.
- Capacity Errors: Уведомление о переполнении полок или достижении лимита шкафов.
- Search Errors: Обработка ситуаций, когда книга не найдена или введен неверный формат GUID.
